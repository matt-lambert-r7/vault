- An initiative around AI on the platform level is already underway. 
- Two tracks: one for more AI assistant type UX. Two, a headless experience where the user can plug in tools like codex, claude code, etc... and interact with the platform
- Gen2 Experience is an internal initiative where designers, PM and engineers are collaborating around these AI problems
- Reviewed existing prototype for AI assistant and learned more about the current thinking at R7 https://drive.google.com/file/d/1agibGTADxNEsEk1MxFZEFk4fTi9nCaYC/view
- The solution to toggle between live chat and previous conversations is a good pattern for a sidebar assistant
- I really like the AI CTA in context idea. Shown on the Alerts by Criticality widget on the Command Home page. This is an excellent shortcut for the user, plus it starts to train them on the power of the chatbot and how to use it. 
- I love seeing the ability to execute actions from inside the chatbot (eg: create a report)
- I do like the idea of the full-ish screen modal for the AI if that is reserved for executing actions. Whether that is generating docs or doing coding/deployment work. 
- I do prefer the AI Summary on the V2 Command Home prototype. Simpler, human-readable, straight forward CTA
- v2 prototype: i like the idea of the agent launcher for targeting specific workflows for the user. Along with the ability to just use the full workbench/chatbot if the user prefers that

### Friction
- Still playing catch up on what thoughts and work are inflight for AI in general
- I like the idea of the widget on the Command Home for what you missed since you were last here, but I don't like how there are competing entry points into engaging with the assistant. 
- I like the idea of the "Requiring your attention" widget on the Command Home page. However, I worry it may conflict with the AI assistant experience. I wonder if they can be combined
- The AI CTA in context of widgets is great, but if it appears in alot of places, does it just start to become noise for the user?
- I like the idea of going to full-ish page modal when a report is generated via AI. However, I wonder if it is too heavy handed. I wonder if we should take a page from Codex here, just generate the document, then inline give the user the choice to view it or not, then pop it open. 
	- This does bring with it the issue of document storage though
	- The notebook idea is interesting
	- I wonder if documents should be stored locally or within the platform?
	- Local storage might make this easier if the documents are generated into markdown
	- Documents would then remain portable
	- Documents would then remain accessible and readable by tools like Codex
- The "Require your attention" list view is overwhelming at first glance. Visual design is awesome, but there is alot of data now fighting for my attention. I wonder if this can be reduced to boost readability and clarity
	- The items in the list all look the same, it is unclear to me which one(s) are the most severe. Perhaps some use of color here would be helpful to distinguish severity?
- Require your attention detail page: Suggested action feels like it should be more prominent. I didn't notice it at first
	- This page actually looks alot like our original investigation design at Kenzo
	- We had one big page with lots of sections
	- We ended up collapsing that into tabs, and introducing "Guided Depth" which was a huge improvement. 
	- That might be an idea to use here... 
		- Step 1 = Overview in 30 seconds
		- Step 2 = little bit more granular evidence or details
		- Step 3 = raw data and deep details
- V2 prototype: the AI action in the header (top right) feels too minimal as just a text field. I preferred the drawer coming out from the right
- V2 prototype: Investigation view: I didn't notice the agent launcher at first. feels like it should be more prominent
- v2 prototype: Investigation view: I like the full-ish window modal for the workbench. However, I lose context of all my investigation attributes when they are covered by the modal.
- v2 prototype: Investigation view: the output of the workbench might not always be a report. It might just be to take an action. 
	- having documents generation happen as inline blocks in the chat might be more widely usable. Similar to how Codex generates docs or updates.
	- User can click a CTA in the block to open the document in a new tab
	- This way the UI can be used for other tasks like executing actions, eg: Close this investigation as a false positive and insert this comment "..."
- Key insights from generated reports: I'd like to see those in the chat as well so I can action on them if I want to. eg: A report identifies a related alert
	  - print out the alert in the chat flow
	  - perhaps a CTA to view the alert or print out more info about it in the chat flow
	  - I want to maintain my focus in the chat, not be taken out into documents that I then need to copy content and paste it back into the chat. Feels like context switching

### Ideas
- AI for SOC Analyst: One of the most important things we could do is to show a SOC Analyst the right questions to ask to reduce noise and address their key tasks/pain points. An open chat field is not enough, guidance on how to prompt is necessary until the user understands who to engage with the chatbot. 
	- Tailored prompt suggestions, based on user role and history (Command Home widget) is an excellent idea
- Instead of the "What you missed" AI widget on Command Home, why not just build this content right into the chatbot when the user first opens it? Can also consider a notifications badge on the "Ask R7" button if an update is present for review
- The "Requiring Your Attention" widget is a great idea. However, I wonder if it can be built into the chatbot experience. I think there is value in funneling as many of these types of actions into a single UI. This will train the user go to this UI and also help to uncover for them how to use it and the power behind it. Items that need attention are not just prompt suggestions though, they would need to be perhaps a widget within the chatbot flow.. perhaps a starting place for when they open it up
- Allow the user to open the Workbench in a new tab, full screen
- v2 prototype: Investigation view: workbench should either display key investigation attributes or hint to the user that they can use the chatbot to reveal those attributes. 
- The workbench is an excellent concept for putting guardrails on a chat UI. However, a more open experience is still needed IMO. This may be where a user plugs in Codex or Claude Code
